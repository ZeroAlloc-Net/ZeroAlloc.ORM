# ZAO063 — [Param(Name = ...)] override is not supported on composite parameters

**Severity:** Error
**Category:** ZeroAlloc.ORM

## Trigger

Fires when `[Param(Name = "...")]` is applied to a parameter whose type
resolves to a **composite** (MultiArgCtor) convention — e.g. a positional
record like `Money(decimal Amount, string Currency)`.

Composite parameters are unpacked positionally by the v0.5 Phase B emit. A
single composite parameter named `total` produces **N** DbParameters, one
per inner ctor argument:

```
@total_Amount
@total_Currency
```

The `[Param(Name = "...")]` override targets a single DbParameter name
and has no way to express "rename each of the N suffixes". Silently
dropping the override (the pre-v0.5 behaviour) would mislead adopters into
shipping a no-op attribute and discovering the issue at runtime via
provider errors like `parameter @custom not found`.

## Fix

Two options:

* **Remove the `[Param(Name = ...)]` override.** Let the generator emit
  `@{paramName}_{ctorArgName}` positionally and update the SQL accordingly.
* **Rename the C# parameter** so the positional naming convention happens
  to match what the SQL expects. The override is for primitive / VO /
  enum parameters only.

## Example

```csharp
public readonly record struct Money(decimal Amount, string Currency);

public sealed partial class Repo(IAsyncDbConnection connection)
{
    // Fires ZAO063 — Name override on a composite parameter.
    [Command("UPDATE Orders SET Amount = @custom_Amount, Currency = @custom_Currency WHERE Id = 1", Kind = CommandKind.NonQuery)]
    public partial Task<int> UpdateTotalAsync([Param(Name = "@custom")] Money total, CancellationToken ct);
}
```

Fix by removing the override and aligning the SQL to the positional
convention:

```csharp
public sealed partial class Repo(IAsyncDbConnection connection)
{
    [Command("UPDATE Orders SET Amount = @total_Amount, Currency = @total_Currency WHERE Id = 1", Kind = CommandKind.NonQuery)]
    public partial Task<int> UpdateTotalAsync(Money total, CancellationToken ct);
}
```

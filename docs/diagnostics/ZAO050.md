# ZAO050 — Nullable composite type requires runtime all-or-nothing check

**Severity:** Warning
**Category:** ZeroAlloc.ORM

## Status

**Active** — fires for every method that uses a nullable composite type at a return position, a parameter position, or as a nested ctor parameter on a FlatRow / DomainEntity.

## What is a nullable composite

A composite is a type with a multi-arg public constructor whose parameters each resolve to a primitive / value-object / enum / single-arg-ctor / static-factory convention (no recursive composites). The canonical example:

```csharp
public readonly record struct Money(decimal Amount, string Currency);
```

A *nullable* composite is the same type used with the `?` annotation:

* `Task<Money?>` (scalar return)
* `record OrderRow(int Id, Money? Total)` (nested in a FlatRow)
* `Task<int> UpdateAsync(Money? total, CancellationToken ct)` (parameter)

## Why does it warn

The composite's underlying SQL columns (`Amount`, `Currency`) are individually nullable on the database side. The C# nullable annotation can't statically prove the schema declares them NOT NULL **together**. The generator therefore enforces the contract at runtime:

* **All composite columns DBNull** → return `null`.
* **Any (but not all) DBNull** → throw `ZeroAllocOrmMaterializationException` with a message naming every column and its DBNull state.
* **Otherwise** → materialize normally.

The mirror image on the bind side (parameter `Money? total`):

* **`total is null`** → every inner `DbParameter` value is `DBNull.Value`.
* **`total` non-null** → standard positional unpacking via `total.Value.Amount` / `total.Value.Currency`.

ZAO050 fires *per nullable composite position* — every method that mentions a nullable composite type produces one diagnostic per occurrence (return position, each affected parameter, each nested ctor-param). For a service that uses `Task<Money?>` across 20 methods, adopters see 20 ZAO050 warnings. This is deliberate: each occurrence is its own runtime all-or-nothing contract surface, and one warning per surface keeps the suppression decision explicit. Adopters who have internalised the contract use **project-level `<NoWarn>ZAO050</NoWarn>`** once and the per-position noise goes away.

## Suppression

If your schema guarantees the composite columns are always populated or always null together (a single NOT-NULL / NULL constraint applied to all of them as a group), the runtime check is a no-op for your data — suppress via one of:

**Project-level (recommended for established repos where the whole schema has been audited).** Single source of truth, silences every occurrence at once:

```xml
<PropertyGroup>
  <NoWarn>$(NoWarn);ZAO050</NoWarn>
</PropertyGroup>
```

**Source-level pragma** (per-method scoping — useful when only a handful of methods use nullable composites and you want the warning to keep firing for new occurrences). Works regardless of `TreatWarningsAsErrors` because ZAO050 always reports at user-source locations (the return-type syntax, the parameter symbol, or the nested ctor-param syntax — never `Location.None`):

```csharp
#pragma warning disable ZAO050
[Query("SELECT Amount, Currency FROM Orders WHERE Id = @id")]
public partial Task<Money?> GetTotalAsync(int id, CancellationToken ct);
#pragma warning restore ZAO050
```

**Visibility without build break** (when you want to see the warning in IDE / logs but not fail the build with `TreatWarningsAsErrors`):

```xml
<PropertyGroup>
  <TreatWarningsAsErrors>true</TreatWarningsAsErrors>
  <WarningsNotAsErrors>$(WarningsNotAsErrors);ZAO050</WarningsNotAsErrors>
</PropertyGroup>
```

All three mechanisms compose. Pick the narrowest that fits your team's policy.

## Diagnostic message

```
Method '<MethodName>' uses nullable composite type '<TypeFullName>' at <position>. The all-or-nothing DBNull check is enforced at runtime; partial-null columns throw ZeroAllocOrmMaterializationException at materialize time (or send DBNull-for-all on bind time). If your schema guarantees these columns are populated or null together, suppress this warning with #pragma warning disable ZAO050.
```

`<position>` is one of:

* `return position` — scalar `Task<Money?>`.
* `nested in return type` — nullable composite inside a FlatRow / DomainEntity ctor.
* `parameter '<paramName>'` — `Money? total` parameter.

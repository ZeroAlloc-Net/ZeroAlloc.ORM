# ZAO044 — Ambiguous convention discovery

**Severity:** Error
**Category:** ZeroAlloc.ORM

## Status

**Active (v0.5 Phase D)** — fires when `[Materialize(Factory = "X")]` resolves to multiple static overloads with the same name. Reserved for additional future ambiguity surfaces (custom convention plugins, partial-source generators, etc.).

## Trigger

Fires today when:

- A `[Materialize(Factory = "Name")]` annotation resolves to **multiple** matching `static public` (or `internal`) methods on the target type — overload selection is non-deterministic, so the generator refuses to pick one silently.

Reserved for future scenarios where a type matches multiple convention rules with equal priority and no clear precedence (e.g. a type carrying both `[ValueObject]` and a hand-rolled `static From` whose semantics conflict).

## Fix

- Reduce to a single `static` overload of the named factory (rename one, or move the others off the type).
- Or supply an explicit, distinct `[Materialize(Factory = "OtherName")]` that resolves unambiguously.

## Example

```csharp
// Fires ZAO044 — two static overloads named 'From':
[Materialize(Factory = "From")]
public sealed record Money(decimal Amount, string Currency)
{
    public static Money From(decimal amount, string currency) => new(amount, currency);
    public static Money From(string amount, string currency) => new(decimal.Parse(amount), currency);
}

// Fix — rename or remove one of the overloads:
[Materialize(Factory = "From")]
public sealed record Money(decimal Amount, string Currency)
{
    public static Money From(decimal amount, string currency) => new(amount, currency);
    public static Money FromString(string amount, string currency) => new(decimal.Parse(amount), currency);
}
```

# ZAO020 — [Query(FromResource)] not yet implemented in v0.1

**Severity:** Info
**Category:** ZeroAlloc.ORM

## Trigger

Fires when a `[Query]` attribute sets `FromResource = true`. In v0.1 the embedded-resource lookup path is deferred to a later milestone, so the `Sql` value is treated as literal inline SQL instead of as a resource name.

## Fix

This is informational — no code change is required. If your `Sql` value happens to also be valid inline SQL, the method will still work. If you actually need the embedded-resource lookup, track the feature in the project backlog and use inline SQL until it ships.

## Example

```csharp
// Emits ZAO020 (informational):
public sealed partial class Repo(IAsyncDbConnection connection)
{
    [Query("SELECT 1", FromResource = true)]
    public partial Task<int> GetOneAsync(CancellationToken ct);  // literal SQL used
}

// Workaround until FromResource ships:
public sealed partial class Repo(IAsyncDbConnection connection)
{
    [Query("SELECT 1")]
    public partial Task<int> GetOneAsync(CancellationToken ct);
}
```

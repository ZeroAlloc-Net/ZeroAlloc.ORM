# ZAO020 — [ORM attribute](FromResource = true) not yet implemented

**Severity:** Info
**Category:** ZeroAlloc.ORM

## Trigger

Fires when a `[Query]` or `[Command]` attribute sets `FromResource = true`. The embedded-resource lookup path is deferred to a later milestone, so the `Sql` value is treated as literal inline SQL instead of as a resource name. The message names the actual triggering attribute (`Query` or `Command`) so the adopter sees a fix that applies to their declaration.

## Fix

This is informational — no code change is required. If your `Sql` value happens to also be valid inline SQL, the method will still work. If you actually need the embedded-resource lookup, track the feature in the project backlog and use inline SQL until it ships.

## Example

```csharp
// Emits ZAO020 (informational) — [Query] variant:
public sealed partial class Repo(IAsyncDbConnection connection)
{
    [Query("SELECT 1", FromResource = true)]
    public partial Task<int> GetOneAsync(CancellationToken ct);  // literal SQL used
}

// Emits ZAO020 (informational) — [Command] variant (v0.4+):
public sealed partial class CommandRepo(IAsyncDbConnection connection)
{
    [Command("INSERT INTO X VALUES (1)", FromResource = true)]
    public partial Task<int> InsertAsync(CancellationToken ct);  // literal SQL used
}

// Workaround until FromResource ships:
public sealed partial class Repo(IAsyncDbConnection connection)
{
    [Query("SELECT 1")]
    public partial Task<int> GetOneAsync(CancellationToken ct);
}
```

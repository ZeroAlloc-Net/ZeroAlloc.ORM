# ZAO041 — No binding strategy resolved for parameter

**Severity:** Error
**Category:** ZeroAlloc.ORM

## Trigger

Fires when a `[Query]` method parameter's type resolves to `ConventionKind.Unknown` — the generator could not find any way to extract a primitive value to assign to `DbParameter.Value`:

1. The type isn't a primitive (int, string, Guid, etc.).
2. The type isn't an enum.
3. The type doesn't carry `[ValueObject]` with a `Value` property.
4. The type doesn't expose a recognised static `From` / `FromValue` factory.
5. The type isn't a single-arg record over a primitive.

If the parameter type fits none of these, the generator has no canonical way to unwrap it for command binding.

## Fix

Pick one of:

- Make the parameter a primitive (int, string, Guid, etc.).
- Annotate the wrapper type with `[ValueObject]` and expose a `Value` property.
- Add a static `From(...)` factory that round-trips through a primitive.
- Convert the parameter type to a record with a single-arg primary ctor over a primitive.

## Example

```csharp
// Fires ZAO041 — no binding strategy:
public sealed class Filter {} // no Value, no factory, no [ValueObject]

public sealed partial class Repo(IAsyncDbConnection connection)
{
    [Query("SELECT 1 FROM Orders WHERE Filter = @filter")]
    public partial Task<int> CountAsync(Filter filter, CancellationToken ct);
}
```

Fix by converting to a value-object wrapper:

```csharp
[ValueObject]
public readonly partial struct Filter
{
    public string Value { get; }
    public static Filter From(string value) => new(value);
}
```

# ZAO033 — SQL statement count exceeds tuple arity

**Severity:** Error
**Category:** ZeroAlloc.ORM

## Trigger

Fires when a `[Query]`-annotated method has a `MultiResultSet` return shape (an N-element tuple inside `Task<...>`) but the SQL contains MORE statements than the tuple has elements. The extra result sets would be silently abandoned by the materializer; the diagnostic forces an explicit decision instead of swallowing query work.

## Fix

Either widen the tuple to capture the extra result set(s), or trim the SQL to match the tuple arity.

## Example

```csharp
// Fires ZAO033 — SQL has 3 SELECTs, tuple only consumes 2:
public sealed partial class Repo(IAsyncDbConnection connection)
{
    [Query("SELECT Id, Total FROM Orders WHERE Id = @id; SELECT Sku, Quantity FROM OrderLines WHERE OrderId = @id; SELECT 1;")]
    public partial Task<(OrderRow Head, List<OrderLineRow> Lines)> GetWithLinesAsync(int id, CancellationToken ct);
}

// Fix A — widen the tuple:
public sealed partial class Repo(IAsyncDbConnection connection)
{
    [Query("SELECT Id, Total FROM Orders WHERE Id = @id; SELECT Sku, Quantity FROM OrderLines WHERE OrderId = @id; SELECT 1;")]
    public partial Task<(OrderRow Head, List<OrderLineRow> Lines, int Marker)> GetWithLinesAsync(int id, CancellationToken ct);
}

// Fix B — trim the SQL:
public sealed partial class Repo(IAsyncDbConnection connection)
{
    [Query("SELECT Id, Total FROM Orders WHERE Id = @id; SELECT Sku, Quantity FROM OrderLines WHERE OrderId = @id;")]
    public partial Task<(OrderRow Head, List<OrderLineRow> Lines)> GetWithLinesAsync(int id, CancellationToken ct);
}
```

# ZAO022 — Return type shape not yet supported

**Severity:** Info
**Category:** ZeroAlloc.ORM

## Status

**Active** — fires when a method's return type passes the outer-shape check
(`Task<T>` / `ValueTask<T>` / `IAsyncEnumerable<T>` / `Task` / `ValueTask`) but
the inner `T` is not one of the materialization shapes the current generator
knows how to emit. Without this informational hint, adopters who try
unsupported shapes (e.g. `Task<List<TRow>>`) see a downstream `CS8795`
"partial method has no implementing body" with no ZeroAlloc-specific guidance.

## Trigger

Fires when the method return type is structurally valid for the ORM surface
(passes ZAO002) but the inner result type isn't materializable by the current
emit pipeline. Common cases:

- `Task<List<TRow>>` / `ValueTask<List<TRow>>` — collection wrappers other
  than `IAsyncEnumerable<T>` are not supported. Use `IAsyncEnumerable<TRow>`
  or a tuple of result sets.
- `Task<TRow[]>` — same reason.
- Unsupported tuple compositions that don't match the multi-result-set
  classification rules.
- Any inner type that fails to resolve a construction strategy (see ZAO040
  for the dedicated diagnostic on that surface).

## Fix

Pick a supported return shape:

- **Single row:** `Task<TRow?>` where `TRow` is a positional FlatRow record,
  a `[Materialize]`-decorated type, a primitive, or a composite.
- **Streaming rows:** `IAsyncEnumerable<TRow>` (add `CancellationToken` with
  `[EnumeratorCancellation]` — see ZAO007).
- **Scalar:** `Task<int>` / `Task<long>` / `Task<string?>` etc. for one of
  the 11 supported primitive scalar types.
- **Multi result sets:** `Task<(TRow1[], TRow2[])>` etc. — see ZAO032 /
  ZAO033 for tuple-arity guidance.

## Example

```csharp
// Fires ZAO022:
public sealed partial class Repo(IAsyncDbConnection connection)
{
    [Query("SELECT id FROM orders")]
    public partial Task<List<OrderRow>> GetOrdersAsync(CancellationToken ct);
}

// Fix — switch to IAsyncEnumerable<T>:
public sealed partial class Repo(IAsyncDbConnection connection)
{
    [Query("SELECT id FROM orders")]
    public partial IAsyncEnumerable<OrderRow> GetOrdersAsync(
        [EnumeratorCancellation] CancellationToken ct);
}
```

## Suppression

ZAO022 ships at `Info` severity so it does not break the build. If you want
to surface it more visibly (or hide it entirely), use the standard MSBuild
levers:

```xml
<PropertyGroup>
  <!-- Treat as warning while you migrate to a supported shape -->
  <WarningsAsErrors>$(WarningsAsErrors);ZAO022</WarningsAsErrors>

  <!-- Or silence it explicitly -->
  <NoWarn>$(NoWarn);ZAO022</NoWarn>
</PropertyGroup>
```

# ZAO005 — Multiple ORM attributes on one method

**Severity:** Error
**Category:** ZeroAlloc.ORM

## Trigger

Fires when a method carries more than one generator attribute. In v0.1 the only shipping attribute is `[Query]`, so this most commonly fires when `[Query]` is applied twice. Future milestones will add `[Command]` and `[StoredProcedure]`; at most one ORM attribute per method is allowed.

## Fix

Remove the duplicate attribute(s) so that exactly one ORM attribute remains on the method.

## Example

```csharp
// Fires ZAO005:
public sealed partial class Repo(IAsyncDbConnection connection)
{
    [Query("SELECT 1")]
    [Query("SELECT 2")]
    public partial Task<int> GetOneAsync(CancellationToken ct);  // duplicate [Query]
}

// Fix:
public sealed partial class Repo(IAsyncDbConnection connection)
{
    [Query("SELECT 1")]
    public partial Task<int> GetOneAsync(CancellationToken ct);
}
```

# ZAO070 — BulkInsert method must take exactly one collection parameter

**Severity:** Error
**Category:** ZeroAlloc.ORM

## Trigger

Fires when a method is annotated with
`[Command(Kind = CommandKind.BulkInsert)]` but the parameter list does
not contain exactly one collection-shaped parameter.

The BulkInsert emit shape walks a single source collection
(`IEnumerable<TRow>`, with `IReadOnlyList<TRow>` preferred for
sized-buffer pre-allocation) and expands the SQL's `VALUES` tuple once
per element. Any other parameter shape — zero collection parameters,
multiple collections, or a scalar parameter that is not a
`CancellationToken` — is statically unsupported.

## Fix

Reduce the parameter list to a single `IReadOnlyList<TRow>` (or
`IEnumerable<TRow>`) parameter plus an optional trailing
`CancellationToken`. Move any other inputs into properties on `TRow`
itself.

```csharp
public sealed partial class Repo(IAsyncDbConnection connection)
{
    // OK — one collection parameter + CT.
    [Command(@"INSERT INTO Orders (Id, Total) VALUES (@Id, @Total)",
        Kind = CommandKind.BulkInsert)]
    public partial Task<int> InsertOrdersAsync(
        IReadOnlyList<OrderRow> rows,
        CancellationToken ct);
}
```

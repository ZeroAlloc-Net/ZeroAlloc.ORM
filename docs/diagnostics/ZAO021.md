# ZAO021 — [Query(Batch = ...)] non-Auto value not yet implemented in v0.1

**Severity:** Info
**Category:** ZeroAlloc.ORM

## Trigger

Fires when a `[Query]` attribute sets `Batch` to anything other than the default `BatchMode.Auto`. BatchMode dispatch (multi-statement batching via `DbBatch`) is deferred to v0.3. In v0.1 the value is silently ignored and queries always use single-command emit.

## Fix

This is informational — no code change is required. The query still executes correctly via a single `DbCommand`. If you need explicit batching, track the feature in the project backlog and use the default mode until it ships.

## Example

```csharp
// Emits ZAO021 (informational):
public sealed partial class Repo(IAsyncDbConnection connection)
{
    [Query("SELECT 1", Batch = BatchMode.Required)]
    public partial Task<int> GetOneAsync(CancellationToken ct);  // Batch ignored in v0.1
}

// Workaround until BatchMode ships:
public sealed partial class Repo(IAsyncDbConnection connection)
{
    [Query("SELECT 1")]
    public partial Task<int> GetOneAsync(CancellationToken ct);
}
```

# ZAO073 — BulkInsert return type must be Task<int> or Task<IReadOnlyList<TIdentity>>

**Severity:** Error
**Category:** ZeroAlloc.ORM

## Trigger

Fires when a `[Command(Kind = CommandKind.BulkInsert)]` method's return
type is anything other than:

* `Task<int>` — the rows-affected sum across the batched insert(s).
* `Task<IReadOnlyList<TIdentity>>` — the identities returned by the
  SQL's `RETURNING` / `OUTPUT INSERTED.*` clause, where `TIdentity` is
  `int`, `long`, `Guid`, or a `[ValueObject]` wrapping one of those.

Any other shape (`Task`, `Task<TRow>`, `IAsyncEnumerable<T>`,
`Task<IReadOnlyList<TRow>>`, etc.) is statically unsupported by the
BulkInsert emit.

## Fix

Pick the shape that matches your SQL:

```csharp
public sealed partial class Repo(IAsyncDbConnection connection)
{
    // Returns SUM(rows-affected) across batched inserts.
    [Command(@"INSERT INTO Orders (Id, Total) VALUES (@Id, @Total)",
        Kind = CommandKind.BulkInsert)]
    public partial Task<int> InsertOrdersAsync(
        IReadOnlyList<OrderRow> rows,
        CancellationToken ct);

    // Returns the OUTPUT INSERTED.Id values, in input order.
    [Command(@"INSERT INTO Orders (Total) OUTPUT INSERTED.Id VALUES (@Total)",
        Kind = CommandKind.BulkInsert)]
    public partial Task<IReadOnlyList<int>> InsertOrdersReturningAsync(
        IReadOnlyList<OrderRow> rows,
        CancellationToken ct);
}
```

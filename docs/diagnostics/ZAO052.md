# ZAO052 — Recursive composite types are not supported in v0.5

**Severity:** Error
**Category:** ZeroAlloc.ORM

## Status

**Active** — fires in v0.5 Phase E.1 when a composite type's constructor contains a parameter whose type is itself a composite (i.e. another multi-arg-ctor type that would be classified as `MultiArgCtor` by convention discovery).

Recursive composites (composite-of-composite) are deferred to v0.6+. The v0.5 positional unpack convention is one-level only: an outer composite `Money(decimal, string)` maps to two SQL columns (or two SQL parameters on the bind side), and that mapping cannot recursively expand a nested composite into a third level. A future release will add multi-level expansion under a SQL-side naming scheme (TBD; candidate: `@total_address_street`).

## Trigger

When the generator classifies a return type or ctor-nested type as a composite candidate AND finds that one of its ctor parameters is itself a composite:

```csharp
public readonly record struct Inner(decimal A, string B);
public readonly record struct Outer(int Id, Inner Inner);   // Outer is recursive — Inner is MultiArgCtor.

public sealed partial class Repo(IAsyncDbConnection connection)
{
    [Query("SELECT Id, A, B FROM Things WHERE Id = @id")]
    public partial Task<Outer> GetOuterAsync(int id, CancellationToken ct);
    // ZAO052: Outer contains a nested composite ctor parameter ('Inner').
}
```

Recursive composites are also detected when the recursive type sits as a ctor parameter inside a `FlatRow` or `DomainEntity` row:

```csharp
public sealed record OrderRow(int Id, Outer O);

[Query("SELECT Id, Id2, A, B FROM Things WHERE Id = @id")]
public partial Task<OrderRow?> GetRowAsync(int id, CancellationToken ct);
// ZAO052: OrderRow.O ('Outer') is recursive.
```

## Fix

Pick one of:

1. **Flatten the inner composite** into the outer ctor — the most common refactor. Replace `Outer(int Id, Inner Inner)` + `Inner(decimal A, string B)` with a single `Outer(int Id, decimal A, string B)`. The SELECT shape is unchanged; the C# type loses one wrapper.

2. **Use `[Materialize(Factory)]` on the outer composite** so the generator dispatches to a user-supplied factory and treats the inner construction as an implementation detail of the factory:

   ```csharp
   [Materialize(Factory = "Build")]
   public readonly record struct Outer(int Id, Inner Inner)
   {
       public static Outer Build(int Id, decimal A, string B)
           => new Outer(Id, new Inner(A, B));
   }
   ```

   The factory's parameters define the SELECT shape (3 columns); the inner composite construction happens inside the factory body. See [the composites cookbook recipe](../cookbook/composites.md) for a worked example.

3. **Drop the inner composite type** if it's not load-bearing — use the underlying primitives directly. Value objects (single-arg-ctor wrappers like `OrderId(int Value)`) are NOT recursive composites and do NOT trigger ZAO052; only multi-arg-ctor inner types do.

## Why not just expand recursively?

The v0.5 design committed to a one-level positional unpack so the SQL-side naming stays predictable:

- **Read side:** the generator reads `reader.GetXxx(ord)` per inner column, one ordinal per ctor parameter. Recursive expansion would require a multi-level column-name scheme (e.g. positional-then-nested) and the column-count math at the outer level would need to know the inner expansion depth.
- **Bind side:** the parameter unpacking emits `@{paramName}_{ctorArgName}` per ctor arg. A recursive composite would need `@{paramName}_{outerArg}_{innerArg}` and the adopter would need to spell the deeper suffixes in their SQL — at which point the abstraction value over hand-rolling the SQL is questionable.

The v0.5 contract: composites are a one-level shape. Adopters who need deeper nesting use `[Materialize(Factory)]` (option 2 above) to keep the SQL author in control.

## Suppression

Not recommended. ZAO052 is an Error and surfaces a real shape that the v0.5 emit cannot lower. Suppressing it would not make the generator produce valid code — the affected method would emit a stub or fail to compile.

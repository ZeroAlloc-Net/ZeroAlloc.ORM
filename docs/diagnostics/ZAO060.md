# ZAO060 — [StoredProcedure] async method has out/ref parameter (reserved)

**Severity:** Error (reserved — not emitted in v0.4)
**Category:** ZeroAlloc.ORM
**Status:** Reserved — no emission in v0.4.

## Status

ZAO060 is **reserved** in the v0.4 catalog and is intentionally never emitted by the current generator.

The case it was originally scheduled for — an `out` or `ref` parameter on an
`async` method annotated with `[StoredProcedure]` — is already rejected by the
C# compiler itself with [CS1988](https://learn.microsoft.com/en-us/dotnet/csharp/misc/cs1988) ("Async methods cannot have ref, in or out parameters"),
so a parallel ZAO060 emit would be dead code today. Reserving the ID number
keeps the catalog stable and leaves room for a future, friendlier diagnostic.

## Future use

A later release may swap ZAO060 in for a guided message that points adopters
at the **named-tuple output pattern** when they reach for `out` / `ref` on a
`[StoredProcedure]` wrapper. For example, instead of CS1988, a future build
could surface:

> ZAO060: '`GetCountAsync`' is async — `out int total` cannot apply. Use a
> named-tuple return: `Task<(int Result, int Total)>` where `Total` matches a
> parameter name and gets bound to the procedure's `@Total` OUTPUT parameter.

The named-tuple output pattern is documented in
[`docs/design/2026-05-30-v1.0-design.md`](../design/2026-05-30-v1.0-design.md)
section 4.5.

# ZAO032 — Tuple arity exceeds SQL statement count

**Severity:** Error
**Category:** ZeroAlloc.ORM

## Trigger

Fires when a `[Query]`-annotated method has a `MultiResultSet` return shape (an N-element tuple inside `Task<...>`) but the SQL contains fewer statements than the tuple has elements. The generator stops emitting the body because the runtime would attempt to advance the reader past the last result set and throw at execution time.

## Fix

Either add the missing `SELECT` statement(s) to the SQL so the count matches the tuple arity, or shrink the tuple to match the number of statements you actually issue.

## Example

```csharp
// Fires ZAO032 — tuple expects 3 result sets, SQL has only 1:
public sealed partial class Repo(IAsyncDbConnection connection)
{
    [Query("SELECT 1")]
    public partial Task<(int A, int B, int C)> GetThreeAsync(CancellationToken ct);
}

// Fix A — add the missing SELECTs:
public sealed partial class Repo(IAsyncDbConnection connection)
{
    [Query("SELECT 1; SELECT 2; SELECT 3")]
    public partial Task<(int A, int B, int C)> GetThreeAsync(CancellationToken ct);
}

// Fix B — drop the tuple arity to match the SQL:
public sealed partial class Repo(IAsyncDbConnection connection)
{
    [Query("SELECT 1")]
    public partial Task<int> GetOneAsync(CancellationToken ct);
}
```

# ZAO007 — IAsyncEnumerable<T> return without [EnumeratorCancellation]

**Severity:** Error
**Category:** ZeroAlloc.ORM

## Trigger

Fires when a `[Query]`-annotated method returns `IAsyncEnumerable<T>` but the method has no `CancellationToken` parameter marked `[EnumeratorCancellation]`. Without that attribute the C# compiler does not forward the consumer-supplied `WithCancellation` token to the iterator, so cancellation cannot propagate into the data reader loop.

The diagnostic message varies depending on which sub-case fires:

- A `CancellationToken` parameter exists but is missing `[EnumeratorCancellation]` — the message ends with `…its CancellationToken parameter lacks [EnumeratorCancellation].`
- No `CancellationToken` parameter at all — the message ends with `…has no CancellationToken parameter.`

The fix advice in both cases is the same: a `CancellationToken` parameter attributed with `[EnumeratorCancellation]` must be present.

> Note: v0.1 does not yet emit `IAsyncEnumerable<T>` bodies (streaming is deferred). This diagnostic ships as a forward-looking guard.

## Fix

Add a `CancellationToken` parameter attributed with `[EnumeratorCancellation]` (from `System.Runtime.CompilerServices`). If the parameter already exists, add the attribute to it; otherwise add both.

## Example

```csharp
// Fires ZAO007:
public sealed partial class Repo(IAsyncDbConnection connection)
{
    [Query("SELECT id FROM t")]
    public partial IAsyncEnumerable<int> StreamAsync(CancellationToken ct);  // missing attr
}

// Fix:
using System.Runtime.CompilerServices;

public sealed partial class Repo(IAsyncDbConnection connection)
{
    [Query("SELECT id FROM t")]
    public partial IAsyncEnumerable<int> StreamAsync([EnumeratorCancellation] CancellationToken ct);
}
```

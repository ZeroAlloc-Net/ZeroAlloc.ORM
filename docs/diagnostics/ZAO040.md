# ZAO040 — No construction strategy resolved for type

**Severity:** Error
**Category:** ZeroAlloc.ORM

## Trigger

Fires when a `[Query]` method's return-type element resolves to `ConventionKind.Unknown` — the generator could not match any of the construction strategies the priority ladder checks:

1. Explicit `[Materialize]` (reserved for v0.5).
2. Primitive (int, string, Guid, etc.).
3. `[ValueObject]`.
4. Static `From` / `FromValue` factory.
5. Single-arg record primary ctor.
6. Enum (with optional `[StoreAsString]`).
7. Multi-arg class ctor (DomainEntity).

If the element type fits none of these, the generator has no way to construct an instance from a row.

## Fix

Pick one of:

- Define a static `From(...)` factory on the type.
- Annotate the type with `[ValueObject]` (and provide `Value` / `From`).
- Convert the type to a record with a single-arg primary ctor over a primitive.
- Add `[Materialize(Factory = "...")]` once v0.5 ships.

## Example

```csharp
// Fires ZAO040 — no construction strategy:
public sealed class Mystery {} // no ctor with params, no factory, no attribute

public sealed partial class Repo(IAsyncDbConnection connection)
{
    [Query("SELECT * FROM Mysteries LIMIT 1")]
    public partial Task<Mystery?> GetAsync(CancellationToken ct);
}
```

Fix by adding a factory or rewriting as a record:

```csharp
public sealed record Mystery(int Id, string Name);
```

# ZAO074 — CommandKind.BulkInsert is ignored on this attribute

**Severity:** Info
**Category:** ZeroAlloc.ORM

## Trigger

Fires when `CommandKind.BulkInsert` is referenced on a method
annotated with `[Query]`, `[StoredProcedure]`, or any other ZeroAlloc.ORM
attribute that exposes a `Kind` property only for symmetry with
`CommandAttribute`. On those attributes the `Kind` value is ignored at
emit time; the diagnostic surfaces the no-op at compile time so adopters
do not ship a misleading attribute.

The BulkInsert pipeline is wired exclusively to `[Command]`. The other
attributes have their own emit shapes (`[Query]` materializes rows,
`[StoredProcedure]` calls a sproc) and cannot expand a `VALUES` tuple
positionally.

## Fix

Either change the attribute to `[Command]`, or drop the
`Kind = CommandKind.BulkInsert` argument.

```csharp
public sealed partial class Repo(IAsyncDbConnection connection)
{
    // Move to [Command] to actually get BulkInsert emit:
    [Command(@"INSERT INTO Orders (Id, Total) VALUES (@Id, @Total)",
        Kind = CommandKind.BulkInsert)]
    public partial Task<int> InsertOrdersAsync(
        IReadOnlyList<OrderRow> rows,
        CancellationToken ct);
}
```

# ZAO006 — Method has multiple CancellationToken parameters

**Severity:** Warning
**Category:** ZeroAlloc.ORM

## Trigger

Fires when a generator-annotated method declares more than one `CancellationToken` parameter. The generator only forwards a single token to the underlying ADO.NET call, and the second one would silently be ignored.

## Fix

Use exactly one `CancellationToken` parameter and position it last in the signature. If you need to merge multiple cancellation sources, do so at the caller via `CancellationTokenSource.CreateLinkedTokenSource`.

## Example

```csharp
// Fires ZAO006:
public sealed partial class Repo(IAsyncDbConnection connection)
{
    [Query("SELECT 1")]
    public partial Task<int> GetOneAsync(CancellationToken outer, CancellationToken inner);
}

// Fix:
public sealed partial class Repo(IAsyncDbConnection connection)
{
    [Query("SELECT 1")]
    public partial Task<int> GetOneAsync(CancellationToken ct);
}
```

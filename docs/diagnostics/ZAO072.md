# ZAO072 — BulkInsert placeholder doesn't match any TRow property

**Severity:** Error
**Category:** ZeroAlloc.ORM

## Trigger

Fires when a `[Command(Kind = CommandKind.BulkInsert)]` method's
`VALUES` tuple references a placeholder (e.g. `@Total`) for which the
element type `TRow` exposes no public property of a matching name
(case-insensitive).

The BulkInsert emit binds each placeholder by **name** against the
collection element type's public surface. If a placeholder cannot be
resolved, the generator has no way to materialize a `DbParameter` value
for it.

## Fix

Either rename the SQL placeholder to match an existing `TRow` property,
or add the missing property to `TRow`. Property matching is
case-insensitive but otherwise exact (no aliasing).

```csharp
public sealed record OrderRow(int Id, decimal Total);

public sealed partial class Repo(IAsyncDbConnection connection)
{
    // OK — @Id and @Total both resolve.
    [Command(@"INSERT INTO Orders (Id, Total) VALUES (@Id, @Total)",
        Kind = CommandKind.BulkInsert)]
    public partial Task<int> InsertOrdersAsync(
        IReadOnlyList<OrderRow> rows,
        CancellationToken ct);
}
```

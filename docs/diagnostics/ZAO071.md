# ZAO071 — BulkInsert SQL must contain exactly one VALUES tuple

**Severity:** Error
**Category:** ZeroAlloc.ORM

## Trigger

Fires when a `[Command(Kind = CommandKind.BulkInsert)]` method's SQL
cannot be parsed into a single `VALUES (@p1, @p2, ...)` tuple. The
diagnostic message includes one of three sub-reasons identifying the
specific failure mode:

- **`no VALUES clause found`** — the SQL contains no `VALUES (...)`
  token at all (e.g. an `INSERT … SELECT` shape). BulkInsert's
  auto-multiplication requires a literal `VALUES` clause to expand.

- **`N row tuples in a single VALUES clause`** — the SQL already contains
  multiple row tuples (e.g. `VALUES (1, 2), (3, 4)`). The generator's
  chunk-multiplication adds the additional tuples at emit time; an
  author-written multi-row VALUES leaves the emit path with no
  unambiguous expansion target.

- **`VALUES clause present but no @placeholders found (literal values
  or unsupported expression?)`** — a single `VALUES (...)` tuple was
  found, but its contents don't match the supported
  `@name, @name, ...` shape (typically literal values like
  `VALUES (1, 2)`, or an expression the regex-based parser can't
  tokenise).

The BulkInsert classifier locates the single `VALUES` clause, extracts
the placeholder list, and rewrites it into an N-fold expansion at emit
time (`VALUES (@p1_0, @p2_0, ...), (@p1_1, ...), ...`).

## Fix

Author the SQL with a single `VALUES (@placeholder, ...)` tuple. If you
genuinely need multiple inserts in one round-trip, split into two
methods.

```csharp
public sealed partial class Repo(IAsyncDbConnection connection)
{
    // OK — exactly one VALUES tuple.
    [Command(@"INSERT INTO Orders (Id, Total) VALUES (@Id, @Total)",
        Kind = CommandKind.BulkInsert)]
    public partial Task<int> InsertOrdersAsync(
        IReadOnlyList<OrderRow> rows,
        CancellationToken ct);
}
```

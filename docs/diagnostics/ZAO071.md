# ZAO071 — BulkInsert SQL must contain exactly one VALUES tuple

**Severity:** Error
**Category:** ZeroAlloc.ORM

## Trigger

Fires when a `[Command(Kind = CommandKind.BulkInsert)]` method's SQL
does not contain exactly one `VALUES (@p1, @p2, ...)` tuple.

The BulkInsert classifier locates the single `VALUES` clause, extracts
the placeholder list, and rewrites it into an N-fold expansion at emit
time (`VALUES (@p1_0, @p2_0, ...), (@p1_1, ...), ...`). Two or more
`VALUES` tuples in the source SQL leave the emit path with no
unambiguous expansion target; zero tuples leave it with nothing to
expand.

## Fix

Author the SQL with a single `VALUES (@placeholder, ...)` tuple. If you
genuinely need multiple inserts in one round-trip, split into two
methods.

```csharp
public sealed partial class Repo(IAsyncDbConnection connection)
{
    // OK — exactly one VALUES tuple.
    [Command(@"INSERT INTO Orders (Id, Total) VALUES (@Id, @Total)",
        Kind = CommandKind.BulkInsert)]
    public partial Task<int> InsertOrdersAsync(
        IReadOnlyList<OrderRow> rows,
        CancellationToken ct);
}
```

# ZAO061 — [StoredProcedure] name is empty

**Severity:** Error
**Category:** ZeroAlloc.ORM

## Trigger

Fires when a `[StoredProcedure]` attribute is applied with an empty, `null`, or whitespace-only procedure name — for example `[StoredProcedure("")]` or `[StoredProcedure("   ")]`. The procedure name is written verbatim to `DbCommand.CommandText` paired with `CommandType.StoredProcedure`, so an empty value would reach the underlying provider and surface as an opaque runtime error (e.g. SQL Server's "Could not find stored procedure ''"). Surfacing at compile time gives the adopter a clear, actionable diagnostic instead of a provider-specific failure at first invocation.

## Fix

Pass a non-empty procedure name to the attribute. The value is provider-routed (SQL Server uses the bare name; Postgres treats it as a `CALL`-able routine name), so use whatever identifier the target database expects.

## Example

```csharp
// Fires ZAO061 — empty literal:
public sealed partial class Repo(IAsyncDbConnection connection)
{
    [StoredProcedure("")]
    public partial Task<int> GetCountAsync(CancellationToken ct);
}

// Fires ZAO061 — whitespace-only is treated the same way:
public sealed partial class Repo(IAsyncDbConnection connection)
{
    [StoredProcedure("   ")]
    public partial Task<int> GetCountAsync(CancellationToken ct);
}

// Fix:
public sealed partial class Repo(IAsyncDbConnection connection)
{
    [StoredProcedure("usp_GetCount")]
    public partial Task<int> GetCountAsync(CancellationToken ct);
}
```

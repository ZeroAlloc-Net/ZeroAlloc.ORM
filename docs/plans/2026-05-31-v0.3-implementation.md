# ZeroAlloc.ORM v0.3 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Ship `0.3.0` — multi-result-set tuple emit via `IAsyncDbBatch` (with `;`-joined fallback), `IAsyncEnumerable<T>` streaming, ZAO032/033 multi-result-set diagnostics. Lights up the v0.1-deferred `[Query(Batch = ...)]` non-Auto values (`ZAO021` info diagnostic flips off when the emit honours the override).

**Architecture:** Two new `EmitShape`s — `MultiResultSet` and `Streaming`. The generator detects them via return-type structure:
- `Task<(THead, List<TLine>)?>` → `MultiResultSet`
- `IAsyncEnumerable<T>` → `Streaming`
Both consume the existing `ConventionDiscovery.Resolve` machinery for per-column / per-parameter type resolution.

The `MultiResultSet` emit branches at runtime on `__conn.CanCreateBatch`: batch path uses `IAsyncDbBatch` + per-statement parameters; fallback uses a single command with `;`-joined SQL and `NextResultAsync`. The `Streaming` emit is a single yield-based async iterator with `[EnumeratorCancellation]` on the CancellationToken.

**Tech Stack:** Same as v0.2. `AdoNet.Async.IAsyncDbBatch` is already in the substrate (shipped in AdoNet.Async v1.x).

**Authoritative reference:** [`docs/design/2026-05-30-v1.0-design.md`](../design/2026-05-30-v1.0-design.md) Section 2 (multi-result-set strategy) + the v0.7-relevant `IAsyncEnumerable` notes.

**Working backlog:** [`docs/plans/za-orm-backlog.md`](za-orm-backlog.md) — v0.3 entry (v0.3-T1 through T4).

**Pace:** ~2 weeks single-developer pace. ~22 tasks across 6 phases. TDD per task with snapshot + compile-smoke + integration tests where applicable.

**Lessons baked in (carried from v0.1 + v0.2):**

1. Snapshot test + compile-smoke test for every new emit shape (caught 3 bugs in v0.1, 0 in v0.2).
2. +15-20% per-task budget for analyzer friction (MA0048, MA0004, MA0006, CA1707).
3. `@`-prefix verbatim safety on every emitted identifier reference — carries to batch-command parameters.
4. No `Location`/`Compilation`/`SyntaxNode`/`ISymbol` in cached generator model types — `EquatableArray<T>` for collections.
5. Bundle related tasks per dispatch but no more than ~5-6 commits per agent run to avoid API overload.

### Task A.2 — `BatchEmitStrategy` resolver

**Files:**
- New: `src/ZeroAlloc.ORM.Generator/Model/BatchEmitStrategy.cs` (enum).
- Modify: `src/ZeroAlloc.ORM.Generator/OrmGenerator.cs` (`ClassifyEmitShape` returns batch decision alongside emit shape).

```csharp
internal enum BatchEmitStrategy
{
    SingleCommand,                  // 1 statement: regular ExecuteReaderAsync
    BatchWithFallback,              // multi-statement + BatchMode.Auto: runtime-branch on CanCreateBatch
    BatchAlways,                    // BatchMode.Always: emit batch path only
    JoinedStatementsOnly,           // BatchMode.Never: ;-joined single command only
}
```

Resolver logic per design Section 2:

```
input: sql + BatchMode (from QueryAttribute.Batch)
output: BatchEmitStrategy

if (statement_count == 1):
    return SingleCommand
if (BatchMode == Auto):
    return BatchWithFallback
if (BatchMode == Always):
    return BatchAlways
if (BatchMode == Never):
    return JoinedStatementsOnly
```

Add to `QueryMethodModel` as a cache-safe field.

Snapshot tests: existing snapshots remain unchanged (single-statement paths still go through `SingleCommand`).

**Commit:** `feat(generator): resolve BatchEmitStrategy per QueryAttribute.Batch + statement count`

## Phase B — `MultiResultSet` emit shape (Days 2-5)

### Task B.1 — Detect tuple-of-result-sets return type

Add `EmitShape.MultiResultSet` to the enum.

**Detection** (in `ClassifyEmitShape`):

```
- Unwrap Task<>/ValueTask<>.
- Unwrap optional outer Nullable<> (Task<(...)?> is common shape).
- Inner type is a tuple (INamedTypeSymbol.IsTupleType).
- Tuple has ≥2 elements.
- Each tuple element is either:
  * A scalar type (resolves to ConventionDiscovery's Primitive/Enum/ValueObject)
  * A List<T>/IReadOnlyList<T>/IEnumerable<T> where T is a positional record (FlatRow shape) or DomainEntity (class with ctor).
- Statement count must match tuple arity (else ZAO032/033).
```

Tests: cover detection for `(OrderRow Head, List<OrderLineRow> Lines)`, `(int Total, List<string> Names)`, etc.

**Commit:** `feat(generator): detect MultiResultSet emit shape from tuple return type`

### Task B.2 — Emit batch path (BatchAlways / BatchWithFallback branch true)

The emit for `BatchAlways` and the `if (CanCreateBatch)` branch of `BatchWithFallback`:

```csharp
await using var __batch = __conn.CreateBatch();
var __cmd0 = __batch.CreateBatchCommand();
__cmd0.CommandText = "SELECT ... FROM Orders WHERE Id = @id";
{
    var __p_id_0 = __cmd0.CreateParameter();
    __p_id_0.ParameterName = "@id";
    __p_id_0.Value = @id;
    __cmd0.Parameters.Add(__p_id_0);
}
__batch.BatchCommands.Add(__cmd0);

var __cmd1 = __batch.CreateBatchCommand();
__cmd1.CommandText = "SELECT ... FROM OrderLines WHERE OrderId = @id";
{
    var __p_id_1 = __cmd1.CreateParameter();
    __p_id_1.ParameterName = "@id";
    __p_id_1.Value = @id;
    __cmd1.Parameters.Add(__p_id_1);
}
__batch.BatchCommands.Add(__cmd1);

await using var __reader = await __batch.ExecuteReaderAsync(@ct).ConfigureAwait(false);

// Materialize tuple element 0 from result set 0:
if (!await __reader.ReadAsync(@ct).ConfigureAwait(false)) return null;
var __head = new global::TestApp.OrderRow(/* per-column emit */);

// Move to result set 1:
if (!await __reader.NextResultAsync(@ct).ConfigureAwait(false))
    throw new global::ZeroAlloc.ORM.ZeroAllocOrmMaterializationException("Expected second result set; got none.");

// Materialize tuple element 1 (List<TLine>) from result set 1:
var __lines = new global::System.Collections.Generic.List<global::TestApp.OrderLineRow>();
while (await __reader.ReadAsync(@ct).ConfigureAwait(false))
{
    __lines.Add(new global::TestApp.OrderLineRow(/* per-column emit */));
}

return (__head, __lines);
```

Parameters bound per `BatchCommand` (the spec name `_0`, `_1` suffix avoids collision when multiple commands reference the same `@param`).

**Commit:** `feat(generator): emit IAsyncDbBatch path for MultiResultSet shape`

### Task B.3 — Emit `;`-joined fallback (JoinedStatementsOnly + BatchWithFallback branch false)

The fallback for `JoinedStatementsOnly` and the `else` branch of `BatchWithFallback`:

```csharp
await using var __cmd = __conn.CreateCommand();
__cmd.CommandText = "SELECT ... FROM Orders WHERE Id = @id; SELECT ... FROM OrderLines WHERE OrderId = @id;";
var __p_id = __cmd.CreateParameter();
__p_id.ParameterName = "@id";
__p_id.Value = @id;
__cmd.Parameters.Add(__p_id);

await using var __reader = await __cmd.ExecuteReaderAsync(@ct).ConfigureAwait(false);

if (!await __reader.NextResultAsync(@ct).ConfigureAwait(false)) throw new ZeroAllocOrmMaterializationException(...);

var __lines = new List<global::TestApp.OrderLineRow>();
while (await __reader.ReadAsync(@ct).ConfigureAwait(false))
    __lines.Add(new global::TestApp.OrderLineRow(/* ... */));

return (__head, __lines);
```

Note: parameters bound ONCE (the joined SQL uses the same `@id` for both statements).

**Commit:** `feat(generator): emit ;-joined fallback for MultiResultSet shape`

## Phase C — `IAsyncEnumerable<T>` streaming emit (Days 6-8)

### Task C.1 — Detect Streaming shape

Add `EmitShape.Streaming` to the enum.

**Detection:**
- Return type is `IAsyncEnumerable<T>` (matches by name + arity, since the test compilation may resolve it as an error type if the BCL assembly isn't fully loaded).
- The CancellationToken parameter must carry `[EnumeratorCancellation]` (already covered by ZAO007).
- T (the element type) resolves via `ConventionDiscovery` (primitive, value-object, FlatRow record, DomainEntity class).

**Commit:** `feat(generator): detect Streaming emit shape from IAsyncEnumerable<T> return`

### Task C.2 — Emit yield-based body

```csharp
public partial async global::System.Collections.Generic.IAsyncEnumerable<global::TestApp.OrderRow> StreamAsync(
    [global::System.Runtime.CompilerServices.EnumeratorCancellation] global::System.Threading.CancellationToken @ct)
{
    var __conn = @connection;
    var __openedHere = __conn.State != global::System.Data.ConnectionState.Open;
    if (__openedHere) await __conn.OpenAsync(@ct).ConfigureAwait(false);
    try
    {
        await using var __cmd = __conn.CreateCommand();
        __cmd.CommandText = "SELECT Id, CustomerId, Total FROM Orders ORDER BY Id";
        await using var __reader = await __cmd.ExecuteReaderAsync(@ct).ConfigureAwait(false);
        while (await __reader.ReadAsync(@ct).ConfigureAwait(false))
        {
            yield return new global::TestApp.OrderRow(
                __reader.GetInt32(0),
                __reader.GetInt32(1),
                __reader.GetDecimal(2));
        }
    }
    finally
    {
        if (__openedHere) await __conn.CloseAsync().ConfigureAwait(false);
    }
}
```

**Critical:** reader + cmd dispose correctly even when the consumer breaks the `await foreach` loop early (the `await using` declarations + the outer `try/finally` ensure cleanup).

`yield return` inside `try/finally` is legal in C# — but it desugars into an async iterator state machine. The generator emits the same shape — Roslyn handles it.

Snapshot + compile-smoke test.

**Commit:** `feat(generator): emit yield-based async iterator for Streaming shape`

## Risk register

| Risk | Mitigation |
|---|---|
| `Microsoft.Data.Sqlite` may not support `CanCreateBatch` in all versions | Verify at fixture init; skip tests with `Skip` reason if not supported |
| `yield return` inside async-iterator `try/finally` desugaring quirks | Match the shape Roslyn generates for hand-written `async IAsyncEnumerable<T>` with `try/finally` (well-tested by .NET BCL itself) |
| Multi-result-set runtime branch (BatchWithFallback) explodes snapshot size | Single snapshot per shape, but the body is ~80 lines; review once per shape |
| `BatchEmitStrategy.JoinedStatementsOnly` (BatchMode.Never) breaks if SQL has `;` inside literal strings | Already mitigated by string-literal-aware splitter (Task A.1) |
| API overload during long agent dispatches (per v0.2 crash) | Cap each agent dispatch at ~5-6 commits; bundle conservatively |

# ZeroAlloc.ORM v0.1 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Ship the foundational `0.1.0-preview` NuGet release — annotated `partial` methods with `[Query]` attribute, FlatRow materialization on positional records, primitive parameter binding, AOT-clean publish, snapshot test rig, integration smoke against Sqlite in-memory.

**Architecture:** Roslyn incremental generator scans for `[Query]`-annotated `partial` methods on `partial` classes with an `IAsyncDbConnection` injected via primary constructor. Emits a sibling `*.g.cs` file completing each partial with the ADO.NET pipeline against AdoNet.Async — EF-style ref-counted connection lifecycle (open-on-execute, close-on-reader-dispose). Zero runtime reflection.

**Tech Stack:** .NET 10, Roslyn `Microsoft.CodeAnalysis.CSharp` 4.13.0 incremental generators, Verify.NET 28+ for snapshot tests, Microsoft.Data.Sqlite for integration smoke, AdoNet.Async 1.x for the IAsyncDbConnection substrate.

**Authoritative reference:** [`docs/design/2026-05-30-v1.0-design.md`](../design/2026-05-30-v1.0-design.md). Every task below cites the relevant design section. Anything contradicting the design needs re-check before action.

**Working backlog:** [`docs/plans/za-orm-backlog.md`](za-orm-backlog.md). Items get crossed out as they ship; new findings get appended.

**Pace:** Single developer, 4 weeks. ~30 tasks. Test-first throughout.

using System.Collections.Immutable;
using Microsoft.CodeAnalysis;
using Microsoft.CodeAnalysis.CSharp;

namespace ZeroAlloc.ORM.Generator.Tests;

internal static class GeneratorHarness
{
    public static GeneratorDriverRunResult RunGenerator(string source)
    {
        var syntaxTree = CSharpSyntaxTree.ParseText(source);

        var references = AppDomain.CurrentDomain.GetAssemblies()
            .Where(a => !a.IsDynamic && !string.IsNullOrWhiteSpace(a.Location))
            .Select(a => MetadataReference.CreateFromFile(a.Location))
            .Cast<MetadataReference>()
            .ToImmutableArray();

        var compilation = CSharpCompilation.Create(
            assemblyName: "TestAssembly",
            syntaxTrees: new[] { syntaxTree },
            references: references,
            options: new CSharpCompilationOptions(OutputKind.DynamicallyLinkedLibrary));

        var generator = new ZeroAlloc.ORM.Generator.OrmGenerator();
        GeneratorDriver driver = CSharpGeneratorDriver.Create(generator);
        driver = driver.RunGeneratorsAndUpdateCompilation(compilation, out _, out _);
        return driver.GetRunResult();
    }
}

```csharp
namespace ZeroAlloc.ORM;

/// <summary>
/// Marks a partial method as a SQL query. The generator emits the materialization
/// pipeline against the containing type's <c>IAsyncDbConnection</c>. See the v1.0
/// design Section 2 for return-type dispatch rules and supported method shapes.
/// </summary>
[AttributeUsage(AttributeTargets.Method)]
public sealed class QueryAttribute(string sql) : Attribute
{
    /// <summary>The SQL fragment, or — when <see cref="FromResource"/> is true — the embedded resource name.</summary>
    public string Sql { get; } = sql;

    /// <summary>
    /// When true, <see cref="Sql"/> names an embedded resource (e.g. <c>"MyApp.Sql.GetOrderById"</c>)
    /// rather than inline SQL. Resource lookup happens at generator time.
    /// </summary>
    public bool FromResource { get; init; }

    /// <summary>
    /// Multi-statement strategy. <see cref="BatchMode.Auto"/> (the default) picks <c>IAsyncDbBatch</c>
    /// when the provider supports it and the SQL has multiple statements; otherwise falls back to
    /// a single command with <c>NextResultAsync</c>.
    /// </summary>
    public BatchMode Batch { get; init; } = BatchMode.Auto;
}

/// <summary>Multi-statement execution strategy for <see cref="QueryAttribute.Batch"/>.</summary>
public enum BatchMode
{
    /// <summary>Generator picks based on SQL statement count and <c>CanCreateBatch</c>.</summary>
    Auto,
    /// <summary>Always use <c>IAsyncDbBatch</c>; throw if <c>CanCreateBatch</c> is false.</summary>
    Always,
    /// <summary>Never use <c>IAsyncDbBatch</c>; always single-command + <c>NextResultAsync</c>.</summary>
    Never,
}
```

```csharp
using Microsoft.CodeAnalysis;
using Microsoft.CodeAnalysis.CSharp.Syntax;
using ZeroAlloc.ORM.Generator.Model;

namespace ZeroAlloc.ORM.Generator;

[Generator(LanguageNames.CSharp)]
public sealed class OrmGenerator : IIncrementalGenerator
{
    private const string QueryAttributeFullName = "ZeroAlloc.ORM.QueryAttribute";

    public void Initialize(IncrementalGeneratorInitializationContext context)
    {
        var queryMethods = context.SyntaxProvider
            .ForAttributeWithMetadataName(
                fullyQualifiedMetadataName: QueryAttributeFullName,
                predicate: static (node, _) => node is MethodDeclarationSyntax,
                transform: static (ctx, _) => TransformMethod(ctx))
            .Where(static m => m is not null)!;

        // Group methods by containing type so we emit one .g.cs per type.
        var grouped = queryMethods.Collect()
            .SelectMany(static (methods, _) =>
                methods.GroupBy(m => m!.ContainingTypeFullName, StringComparer.Ordinal)
                       .Select(g => new QueryRepositoryModel(g.Key, g.Cast<QueryMethodModel>().ToImmutableArray())));

        context.RegisterSourceOutput(grouped, EmitRepository);
    }

    private static QueryMethodModel? TransformMethod(GeneratorAttributeSyntaxContext ctx)
    {
        if (ctx.TargetSymbol is not IMethodSymbol method) return null;
        if (method.ContainingType is not INamedTypeSymbol containing) return null;

        var sql = ctx.Attributes
            .FirstOrDefault()?
            .ConstructorArguments
            .FirstOrDefault().Value as string ?? string.Empty;

        return new QueryMethodModel(
            MethodName: method.Name,
            ContainingTypeFullName: containing.ToDisplayString(SymbolDisplayFormat.FullyQualifiedFormat),
            ContainingTypeName: containing.Name,
            Namespace: containing.ContainingNamespace.IsGlobalNamespace
                ? null
                : containing.ContainingNamespace.ToDisplayString(),
            Sql: sql);
    }

    private static void EmitRepository(SourceProductionContext context, QueryRepositoryModel repo)
    {
        var sb = new StringBuilder();
        sb.AppendLine("// <auto-generated/>");
        sb.AppendLine("#nullable enable");
        sb.AppendLine();
        if (repo.Methods[0].Namespace is { } ns)
        {
            sb.AppendLine($"namespace {ns};");
            sb.AppendLine();
        }
        sb.AppendLine($"partial class {repo.Methods[0].ContainingTypeName}");
        sb.AppendLine("{");
        foreach (var m in repo.Methods)
        {
            sb.AppendLine($"    // TODO: emit body for {m.MethodName} -- v0.1 Task 4.x");
        }
        sb.AppendLine("}");

        var hint = $"{repo.Methods[0].ContainingTypeName}.g.cs";
        context.AddSource(hint, SourceText.From(sb.ToString(), Encoding.UTF8));
    }
}
```

`Model/QueryMethodModel.cs`:

using System.Collections.Immutable;

namespace ZeroAlloc.ORM.Generator.Model;

internal sealed record QueryMethodModel(
    string MethodName,

## Quality bars enforced throughout

- **TDD discipline.** Write the test that proves the next feature is missing, watch it fail, implement the minimum to make it pass, watch it pass, commit. Do NOT batch implementations across tasks.
- **No reflection in emit.** Every line the generator emits must trace to a compile-time-known type or symbol. If you're typing `Type.GetType(...)` or `MakeGenericType(...)`, stop and reconsider.
- **Globally-qualified names in emit.** `global::System.Threading.CancellationToken` everywhere. Consumer `using` directives are not the generator's concern.
- **`__`-prefixed locals in emit.** Avoids collision with user parameter names (`id`, `customerId`, etc.).
- **`ConfigureAwait(false)` everywhere.** Library-level discipline.
- **Snapshot tests for every emit shape.** If a new code shape emits, a new `.verified.txt` snapshot tracks it. Diffs become PR-reviewable.
- **One Verify snapshot per test method.** Don't bundle multiple shapes in one snapshot — it makes diff review brutal.
- **Commit per task.** Each numbered task above ends in a single commit. If a task feels too big for one commit, split it before starting.
